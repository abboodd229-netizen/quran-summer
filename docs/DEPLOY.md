# دليل النشر على Railway

التطبيق يُنشر كخدمة واحدة: الخادم يقدّم الـ API و SSE وملفات الواجهة المبنية معًا.

## ⚠️ الأهم: قرص دائم (Volume) لقاعدة البيانات
قاعدة البيانات SQLite ملف على القرص. **بدون قرص دائم ستُفقد كل البيانات عند كل
إعادة نشر/تشغيل.** يجب إضافة Volume وربط مسار القاعدة به.

أيضًا: استخدم **نسخة واحدة فقط** من الخدمة (Replicas = 1). SQLite لا يناسب عدة نسخ
تكتب على نفس الملف.

---

## الخطوات

### 1) أنشئ المشروع والخدمة
- اربط مستودع Git يحتوي هذا المشروع، أو ادفعه إلى Railway.
- Railway سيكتشف Node عبر Nixpacks تلقائيًا.

### 2) أضف القرص الدائم (Volume)
- من إعدادات الخدمة: **+ Volume**.
- Mount path مقترح: `/data`.

### 3) متغيّرات البيئة (Variables)
```
NODE_ENV=production
PORT=8080
SESSION_SECRET=<قيمة-عشوائية-طويلة>
SESSION_TTL_DAYS=7
DATABASE_PATH=/data/app.sqlite
ADMIN_USERNAME=admin
ADMIN_PASSWORD=<كلمة-مرور-قوية>
ADMIN_NAME=مدير النظام
BACKUP_ENABLED=true
BACKUP_KEEP=14
```
> `DATABASE_PATH=/data/app.sqlite` يطابق مسار القرص الدائم. النسخ الاحتياطي يُحفظ في
> `/data/backups` تلقائيًا (ضمن نفس القرص).

### 4) أوامر البناء والتشغيل
ملف `railway.json` المرفق يحدّدها، أو اضبطها يدويًا:
- **Build**: `npm install && npm run build`
- **Start**: `npm start`

> `npm run build` يبني الحزمة المشتركة ثم الواجهة ثم الخادم بالترتيب الصحيح.
> الخادم يقدّم الواجهة من `client/dist` تلقائيًا في الإنتاج.

### 5) التهيئة لمرة واحدة (البذور)
بعد أول نشر ناجح، شغّل البذور مرّة واحدة لإضافة الطلاب الـ204:
- عبر Railway CLI: `railway run npm run seed`
- أو من لوحة Railway: نفّذ الأمر `npm run seed` في الخدمة.

> المجموعات/الحلقات/الأسابيع وحساب المدير تُنشأ تلقائيًا عند الإقلاع حتى بدون بذور،
> لكن الطلاب يحتاجون البذور أو الاستيراد من Excel داخل «الإعدادات».

### 6) النطاق
فعّل Domain من Railway. التطبيق متاح على الجذر `/`، والـ API تحت `/api`.

---

## بعد النشر
1. ادخل بحساب المدير وغيّر كلمة المرور فورًا.
2. أنشئ حسابات المساعدين.
3. تحقّق من ظهور الحلقات والطلاب.

## النسخ الاحتياطي والاستعادة
- نسخ يومي تلقائي إلى `/data/backups/app-<وقت>.sqlite` (يُحتفظ بآخر `BACKUP_KEEP`).
- للاستعادة: أوقف الخدمة، استبدل `/data/app.sqlite` بنسخة احتياطية، أعد التشغيل.
- لتنزيل نسخة: استخدم Railway CLI للوصول إلى القرص.

## ملاحظات
- التحديثات/إعادة النشر آمنة طالما القرص الدائم مربوط (البيانات تبقى).
- لا تستخدم أكثر من نسخة واحدة للخدمة.
